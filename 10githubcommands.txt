10 git commands which have not been used in the task and their description

1. git clone: this command is used to make a copy of a repository from an existing URL. It allows to create a local copy of that repository on local directory from the repository URL.

2. Git Branch Command: This command lists all the branches available in the repository 
 syntax: $git branch

3. Git Merge Command: This command is used to merge the specified branches history into the current branch.
 syntax: $ git merge BranchName

4. Git log Command: This command is used to check the commit history.
 syntax: $ git log

5. Git remote Command: This command is used to connect your local repository to the remote server. It allows to create, view and delete connections to other repositories.

6. Git Init command: This command is used to create a local repository
  syntax: $git init Demo

7. Git add command: this command is used to add one or more files to staging area.
  syntax: $git add Filename

8. Git status command: The status command is used to display the state of the working directory and the staging area.
  syntax: $git status 

9. Git push Command: It is used to upload local repository content to a remote repository. Pushing is an act of transfer commits from your local repository to a remote repo.
  syntax: $ git push [variable name] master  

10. Git pull command: Pull command is used to receive data from GitHub. It fetches and merges changes on the remote server to your working directory.
  syntax: $ git pull URL
